# NORTH_STAR — 백워드 플래닝 프롬프트 (knowledge-ops)
### 목적: 매 세션 시작 시 붙여, "최선의 최종결과"에서 역산해 드리프트 없이 가장 큰 레버리지로 움직이게 한다.

---

## A. 고정 맥락 (바꾸지 않음)
- **궁극 목표:** 데이터 분석가 신입 취업. 차별점 = "AI/LLM로 실제 비즈니스 데이터 문제를 푸는 분석가".
- **이 플래그십의 역할:** 그 차별점을 *채용담당자가 60초에 납득하는 산출물*로 증명.
- **잠금 문제정의:** `docs/PROBLEM_DEFINITION.md` (코퍼스·분석질문 Q1~Q5). **원칙:** `CLAUDE.md`(분석단위 분리·AI투명성·의사결정 종료·스코프 가드).
- **결정 이력:** `docs/DECISION_LOG.md` (D-001~D-010). 피벗·판단은 면접 자산.

## B. 현재 단계 (갱신: 2026-06-24)
- Phase 0 문제정의 잠금 ✅
- Phase 1 개인 코퍼스(GitHub stars): 수집→프로파일 EDA→지식그래프→문제→소환 ✅ *(파이프라인·진정성 검증 완료, 동결)*
- Phase 3 조직 코퍼스(GitLab Handbook · People/Hiring): 구조분석 스크립트 준비완료, **실행 직전** ⏳
- Phase 4 포트폴리오화(내러티브·시각화·면접 STAR·이력서 연결) ❌ 미착수
- 진척: 기술 파이프라인 ~75% / 취업가치(패키징) ~25%

## C. 최선의 최종결과 (이걸로 역산)
> 단 하나의 날카로운 포트폴리오:
> **"LLM로 조직 지식베이스(GitLab Handbook)를 구조화·진단해 공백·사일로·중복을 찾고 '무엇을 먼저 문서화/통합/폐기'할지 우선순위를 제시하며, 특정 문제(온보딩)에 필요한 지식을 소환한다."**

좋은 결과의 5조건:
1. **비즈니스 문제 → 결정**으로 끝난다(진단만으로 끝나지 않음).
2. 숫자는 **결정적·재현 가능**(구조지표), LLM 추출은 **샘플 휴먼검증**으로 신뢰 입증.
3. **AI가 한 것 vs 내가 판단한 것**이 분리돼 면접에서 판단력 스토리가 된다(D-006/D-010 류).
4. 채용담당자가 **60초**에 이해(1장 내러티브 + 시각화 1개).
5. 스코프가 좁다(코퍼스 2개·결정 1~2개). 노트·기능 무한확장 없음.

## D. 매 행동 판정 규칙 (이 프롬프트의 본체)
어떤 작업을 제안/실행하기 전에 4질문에 답하고, 통과 못 하면 하지 않는다:
1. **레버리지:** 이게 C(최선결과)에 직접 기여하나? 아니면 파이프라인 광택(드리프트)인가?
2. **취업 환산:** 이 작업이 *이력서 한 줄 / 면접 한 답변 / 포트폴리오 한 장*으로 떨어지나?
3. **종결성:** "그래서 무슨 결정"으로 끝나나?
4. **스코프:** 코퍼스·기능을 늘리지 않나? 늘린다면 D-0xx로 명시 기록했나?

## E. 출력 형식 (세션마다 이 순서로 답)
1. **현 위치** — B를 1~2줄로 갱신.
2. **최선결과까지의 격차** — C 대비 무엇이 비었나(특히 Phase 4).
3. **다음 1~2 행동** — 가장 큰 레버리지 순으로, D 규칙 통과 근거와 함께. (메뉴 나열 금지, 추천 1개 + 대안 1개)
4. **드리프트 점검** — 지금 하려는 게 과거 실패모드(수단이 목적 잡아먹기)인지 한 줄 자기검열.

## F. 지금 시점의 적용 (예시 답)
- 다음 최대 레버리지: **Phase 3을 린하게 종료**(핸드북 진단 결정 1개 + LLM 개념추출 샘플검증) → 곧장 **Phase 4 착수**(1장 내러티브·시각화·면접 STAR). 
- 하지 말 것: 코퍼스 추가, 그래프 알고리즘 고도화, 새 도구 — 전부 D 규칙 1·2에서 탈락.
