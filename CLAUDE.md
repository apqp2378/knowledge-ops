# CLAUDE.md — knowledge-ops 작업 지침

## 프로젝트 한 줄
조직 지식이 의사결정 순간 안 쓰이는 문제를 AI로 진단한다. 산출물은 "분석 + 의사결정"이지, 노트앱·도구가 아니다.

## Karpathy 4원칙
1. 코딩 전에 생각: 추측 금지, 해석 여러 개면 선택지 제시.
2. 최소한의 코드: 요청 외 추가 금지. 100줄로 되면 1000줄 짜지 말 것.
3. 수술적 수정: 요청한 부분만. 수정 후 "요청한 것만 변경" 명시.
4. 성공 기준 기반: "완료" 대신 검증 결과 보고.

## KoD 추가 원칙
- **분석 단위 분리:** 영상 1건 / 개념(노드) 1건 / 관계(엣지) 1건 / 문제 시나리오 1건을 혼동하지 않는다. 함수 입출력 단위를 docstring에 명시.
- **AI 투명성:** LLM이 추출·분류한 것과 내가 검증·판단한 것을 명확히 구분한다. LLM 분류(신뢰등급 등)는 **샘플 휴먼검증으로 정확도를 입증**한다 — 입증 없는 LLM 출력은 결론에 쓰지 않는다.
- **의사결정 종료:** 모든 분석은 "그래서 무슨 결정"으로 끝난다. 결정 없는 분석은 미완.
- **스코프 가드:** 산출물은 의사결정 1~2개로 한정. 노트 무한생산·기능 무한확장 금지. Obsidian/저장도구는 수단, 평가 대상은 분석.
- **운영 보조 지표:** 신뢰등급·공백·합의도는 절대 점수가 아니라 상대 비교·의사결정 보조다.

## 권한(MCP) 원칙
도구 권한 기본 "승인 필요". 워크스페이스/레포 단위 분리. 민감정보 평문 보관 금지.
